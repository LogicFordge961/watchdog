Core features

Crash grouping

Automatically groups similar errors by stack trace and context so you can prioritize the root cause instead of duplicated noise.

Helpful diagnostics

Collects environment, logs, breadcrumbs, and minimum reproduction details to accelerate triage and debugging.

Privacy-first

Sanitization hooks and opt-in attachments let you control which data is collected and shared with support teams.

How it works

1
Install the lightweight agent

Drop the small WatchDog runtime into your app; it has minimal memory and performance impact.

2
Capture events & context

On crashes, WatchDog collects a compact diagnostic bundle (stack, app state, breadcrumbs).

3
Get actionable reports

Reports are grouped, prioritized, and can be forwarded to Slack, GitHub, or email for fast resolution.

Quick start

  1. Download the agent from the Download page.
  2. Drop it into your app and call WatchDog.init() with your key.
  3. Reproduce an issue and check the Support inbox or dashboard for reports.
Need help? Submit a detailed report on Support and attach logs/screenshots.

Frequently asked

Is WatchDog safe to use in production?

Yes — it's designed for production use. You can enable sampling and configure what data is collected to meet your privacy needs.

Does it upload user data by default?

No sensitive user data is uploaded by default. Hooks let you sanitize or remove fields before a report is sent.

What platforms are supported?

Windows, macOS, and Linux are supported today. Mobile and embedded platforms are planned — check the roadmap on GitHub.
